🧩 架构分层

UI 层 — app/ + components/

  • expo-router 文件路由（Tab 导航 + Modal）
  • 8个可复用组件，涂鸦风设计
  • BigPoopButton: spring动画 + 缩放旋转
  • CharacterDisplay: 心情状态动画（开心弹跳/难过下雨）

状态层 — lib/store.ts (Zustand)

  • 全局 Store：用户/角色/记录/社区/UI 状态
  • Actions：addRecord、updateCharacter、setUser…
  • Computed：getTodayRecords、getCurrentStreak
  • 打卡自动更新 EXP + lastCheckIn

业务逻辑层 — lib/character.ts

  • 5级进化：😖💩 → 😊💩 → 😄💩🎩 → ✨💩👑 → 🌈💩🪽
  • 纯 EXP 升级制（0/7/30/90/150 EXP）
  • 连续打卡 bonus：3天+1, 7天+2, 30天+3, 60天+5
  • 角色心情：happy(2h内) → normal(24h) → anxious(3天) → sad(3天+)

基础设施层 — Supabase

  • Auth：用户注册/登录
  • Database：users / poop_records / posts / comments / likes
  • ⚠️ 尚未接入（env key 待配）

🔄 核心流程：排便打卡

💩 大按钮 → Modal 弹窗 → Bristol 分型 → 心情选择 → 备注(可选) → 提交
store.addRecord() → EXP + bonus → 角色升级检查 → 庆祝动画 🎉

📋 数据模型

PoopRecord

  • id user_id bristol_type(1-7)
  • mood(terrible/okay/good/amazing) note
  • created_at date(YYYY-MM-DD)

User

  • character_level character_exp
  • current_streak max_streak total_records
  • achievements[]

Post（树洞）

  • type(🎉成就晒/😫吐槽/💡小妙招/💩打卡分享)
  • is_anonymous 匿名发帖
