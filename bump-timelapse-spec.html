📸 孕肚时光机

BumpLapse · MLP Product Spec v1.0

每周拍一张孕肚照，自动变成让你哭的延时视频。 没有注册、没有社区、没有广告、没有后端。照片不离开你的手机。

📋 产品概要

产品名孕肚时光机 / BumpLapse
一句话每周拍一张孕肚照，自动生成延时视频
目标用户孕 12-40 周的准妈妈（核心）+ 准爸爸（辅助拍摄）
平台iOS only（MLP 阶段）
商业模式免费增值（Freemium）：免费拍+存，付费导出无水印视频
竞品几乎没有专做这个的 app，现有方案是手动相册整理或通用视频剪辑
核心差异① 拍照引导对齐 ② 一键延时视频 ③ 零后端零隐私顾虑

用户画像

妙子，28岁，孕 24 周，坐标苏州

  • 每周都想拍孕肚照发朋友圈，但角度总是不一样，回头看很乱
  • 在小红书看到别人的孕肚延时视频，非常想要但不会剪辑
  • 试过让老公帮忙拍，每次站位不同，拼起来效果差
  • 不想下载美柚那种大而全的 app，只想要一个简单工具
  • 对隐私敏感——孕肚照不想上传到任何服务器

🔄 核心用户流程

📱 打开 App → 📸 全屏相机 → 👤 对齐引导拍照 → ✅ 存入时光轴 → ✨ 生成视频 → 💬 分享

关键约束

  • 3 秒开拍：打开 app → 相机界面，中间不要任何阻断
  • 首次无注册：第一次打开只问预产期（1步），然后直接进相机
  • 延迟登录：只有导出视频时才需要 Apple ID 登录（走 IAP 付费）
  • 每周提醒：可设置每周固定时间推送提醒拍照

首次打开流程

1️⃣ 欢迎页（1屏）：产品 slogan + "开始记录" 按钮

2️⃣ 输入预产期（日期选择器，1屏）→ 自动算出当前孕周

3️⃣ 设置每周提醒时间（可跳过）

4️⃣ → 直接进入相机拍第一张

总共不超过 30 秒。不要邮箱、不要手机号、不要密码。

🛠️ 技术方案

架构：纯端侧，无后端

层方案说明
UISwiftUIiOS 16+，纯 SwiftUI
相机AVFoundation自定义相机界面，支持前后置切换、倒计时
姿势检测Vision FrameworkVNDetectHumanBodyPoseRequest，检测身体中心点，引导对齐
照片存储本地沙盒 + PhotosApp 沙盒内存原图，同时存入系统相册
云同步iCloud (CloudKit)换手机不丢数据，苹果原生方案，不经过我们的服务器
视频合成AVFoundationAVMutableComposition + AVVideoCompositionCoreAnimationTool
图像对齐Vision + CIFilter检测人体中心点，自动裁剪对齐，CIFilter 做过渡
BGM内置 .m4a3 首免版权音乐，AVAudioPlayer 混合
付费StoreKit 2 + RevenueCatIAP 管理，非订阅型（一次性购买）
推送UNUserNotificationCenter本地推送，每周提醒拍照，不需要推送服务器
分析TelemetryDeck隐私友好的匿名分析（替代 Firebase）

隐私设计（核心卖点）

  • 📵 零网络请求（除 IAP 和 iCloud 同步）
  • 🔒 照片不离开设备，不上传到任何第三方服务器
  • 🍎 iCloud 同步是用户自己的 Apple ID，我们无法访问
  • 📊 分析只收集匿名事件（打开次数、拍照次数、生成视频次数），不收集个人信息
  • App Store 隐私标签：Data Not Collected
